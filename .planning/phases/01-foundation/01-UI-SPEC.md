# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | lucide-react |
| Font | Geist (Next.js default — clean sans-serif) |

**Rationale:** Greenfield project. Tailwind CSS for styling, no component library. Lucide-react for generic icons (users, copy, arrow-right, etc.). Piece rank icons are custom inline SVG glyphs (stars, bars) per PROJECT_SPECS.md.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element spacing, piece icon padding |
| md | 16px | Default element spacing, button padding |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps, board cell padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, vertical centering |

**Exceptions:** Board cells use 40px fixed width/height to maintain square aspect ratio regardless of content. Piece icons sized at 24px within cells. The 8-point scale applies to all other spacing.

## Components

### 1. Landing Page (`/`)

- Full-page centered layout with page title + subtitle
- Two stacked CTA buttons: "Play vs Bot" (accent) and "Play Online" (secondary/outline)
- Buttons: full-width on mobile (max-w-sm), auto-width on desktop
- Subtle background: dominant color with slight radial gradient for depth

### 2. Lobby Page (`/lobby`)

- Two-tab or two-panel layout: "Create Room" and "Join Room"
- **Create panel:** name input + bot toggle + "Create" button
- **Join panel:** name input + room code input (uppercase, 6-char) + "Join" button
- **Waiting state:** room code display (large, monospace), player list, "Leave" button
- Card-based layout, vertically centered

### 3. Game Page (`/game/[roomId]`)

#### Board Component
- 9-column × 8-row CSS Grid
- Cell size: 48px × 48px desktop, 36px × 36px mobile
- Alternating light/dark green squares
- Deployment zone overlays (red rows 0-2, blue rows 5-7) — subtle tint, not full overlay
- Board border: 2px solid `secondary` color

#### Piece Component
- Circular or rounded-square container
- Player color as background (red/blue)
- White rank icon centered (SVG stars for generals, bars for officers, single symbols for others)
- Size: fills cell minus 4px padding
- **Deployed piece:** shows icon + rank abbreviation
- **Undeployed piece (palette):** icon + count badge

#### Piece Palette Component
- Horizontal scrollable row of available pieces for deployment
- Grouped by rank type
- Each item: piece preview + count remaining
- Click to select, click board square to place
- Selected state: gold border

#### Deployment Zone Indicator
- Subtle tinted overlay on rows 0-2 (red) and 5-7 (blue)
- Disappears once game phase starts

#### Game Header
- Shows: room code, player names + colors, current phase/status
- Fixed at top of game area

## Interaction Patterns

| Action | Feedback |
|--------|----------|
| Click piece in palette | Gold border highlight, cursor changes to crosshair or pointer |
| Click invalid board square (deployment) | Brief shake animation or red flash on cell |
| Click valid board square (deployment) | Piece appears on board with subtle pop-in |
| Hover board square | Subtle brightness increase |
| Click Ready | Button state → "Ready ✓", opponent sees "Opponent Ready" indicator |
| Both ready | Brief transition to playing phase |
