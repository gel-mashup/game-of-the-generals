    Add CSS board flip to Board.tsx. The board grid rotates 180° when playerSide === 'red' so the red deployment zone (rows 0-2) appears at the bottom of the screen.

    1. On the grid container div (the one with `grid grid-cols-9 grid-rows-8`), add conditional rotation:
       - Current: `<div className="relative grid grid-cols-9 grid-rows-8 gap-0 aspect-[9/8] border-4 border-[#1e3a5f] rounded-lg overflow-hidden shadow-2xl">`
       - New: `<div className={`relative grid grid-cols-9 grid-rows-8 gap-0 aspect-[9/8] border-4 border-[#1e3a5f] rounded-lg overflow-hidden shadow-2xl transition-transform duration-500 ease-in-out ${playerSide === 'red' ? 'rotate-180' : ''}`}>`
       - `rotate-180` is a standard Tailwind utility (`transform: rotate(180deg)`)
       - `transition-transform duration-500 ease-in-out` provides smooth rotation animation

    2. Inside each cell div, wrap the Piece render in a counter-rotation div:
       - Current: `{piece && (<Piece piece={piece} position={{ row: rowIndex, col: colIndex }} ... />)}`
       - New: `{piece && (<div className={playerSide === 'red' ? 'rotate-180' : ''}><Piece piece={piece} position={{ row: rowIndex, col: colIndex }} ... /></div>)}`

    3. The counter-rotation keeps piece text (5★, Col, Spy, etc.) right-side-up. 180° + 180° = 360° = 0° visual rotation.

    4. Do NOT modify: data-row/data-col attributes, click handlers, cell background colors, valid move highlighting. CSS transform is purely visual — DOM structure and event targets are unchanged.

    5. playerSide comes from `useRoomStore()` which is already imported in Board.tsx.
