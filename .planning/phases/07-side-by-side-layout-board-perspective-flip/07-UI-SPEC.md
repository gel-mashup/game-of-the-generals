## Board Flip Contract

### Transform

```css
transform: rotate(180deg);
```

Applied to the board grid container when `playerSide === 'red'`.

### Rules

1. Board always flipped during both `deploying` and `playing` phases
2. Flip is visual-only — no data model changes, no coordinate remapping in store
3. Click coordinate resolution handled by Board component or `useBoardTransform` hook:
   - When flipped: visual click at (screenX, screenY) → map to logical (row, col) accounting for 180° rotation
   - Formula: `logicalRow = 7 - visualRow`, `logicalCol = 8 - visualCol`
4. Piece text/icons inside cells ALSO rotate 180° to remain readable:
   - When board is rotated 180°, each cell's content must counter-rotate with `transform: rotate(180deg)` so text is not upside-down
5. Deployment zone overlay must track the flipped board (visual bottom = logical top for red player)

### Coordinate Mapping

| playerSide | Visual Bottom Rows | Logical Rows | Deploy Zone |
|------------|-------------------|--------------|-------------|
| blue | 5-7 (logical) | 5-7 | rows 5-7 (bottom by default) |
| red | 0-2 (logical, shown at bottom via flip) | 0-2 | rows 0-2 (shown at bottom via flip) |

## Piece Palette Contract

### Structure

Vertical list in sidebar, grouped by rank tier. Always expanded (no collapsible sections).

### Tier Groups

| Tier | Header Label | Pieces |
|------|-------------|--------|
| Generals | `Generals (5★)` | 5-Star General (×1), 4-Star General (×1) |
| Officers | `Officers (4-2★)` | 3-Star General (×1), 2-Star General (×1), 1-Star General (×1), Colonel (×1) |
| Special | `Special` | Lieutenant (×2), Spy (×2) |
| Privates | `Privates (PVT)` | Private (×6) |

### Piece Item Layout

```
┌──────────────────────────────────┐
│ [symbol]  Piece Name         ×3 │
└──────────────────────────────────┘
```

- Horizontal layout: icon (left) | name (flex-1) | count badge (right)
- Selected piece: `ring-2 ring-[#d4a847] bg-white/10`
- Available pieces: `bg-white/5 hover:bg-white/10`
- Fully deployed pieces: `opacity-40 cursor-not-allowed`
- Item padding: `px-3 py-2` (12px horizontal, 8px vertical)
- Gap between icon and name: `gap-2` (8px)

### Piece Counter Badge

- Shows remaining count: `×N` where N = total - deployed
- When N = 0: badge hidden, item dims to `opacity-40`
- Font: `text-xs font-semibold text-gray-300`

### Sidebar Header

- Title: "Deploy Your Forces" — `text-lg font-bold text-[#d4a847]`
- Progress: "N/21 pieces placed" — `text-sm text-gray-400`

### Buttons (at sidebar bottom)

| Button | Style | States |
|--------|-------|--------|
| Auto-Deploy | `bg-gray-600 hover:bg-gray-500 text-white font-medium rounded-lg` | Disabled when `playerReady` |
| Ready | `bg-[#d4a847] hover:bg-[#c49a3f] text-white font-bold rounded-lg` | Disabled when not all deployed or already ready |
| Ready ✓ | `bg-green-700 text-white font-bold rounded-lg cursor-default` | Terminal state after clicking Ready |

## Glass-Morphism Sidebar Contract

### Visual Spec

| Property | Value |
|----------|-------|
| Background | `rgba(30, 58, 95, 0.5)` (navy 50% opacity) |
| Backdrop filter | `backdrop-blur-md` (~12px blur) |
| Border | `border-l border-white/10` (subtle left edge) |
| Border radius | `rounded-l-lg` (left corners only, 8px) |
| Shadow | `shadow-2xl shadow-black/30` |
| Overflow | `overflow-y-auto` (scrollable if content exceeds height) |
| Z-index | `z-30` (above board, below modals) |

### Show/Hide Transition

| Property | Value |
|----------|-------|
| Transition | `transition-transform duration-300 ease-in-out` |
| Visible state | `translate-x-0` |
| Hidden state | `translate-x-full` |

### Mobile Fallback (< 768px)

- Sidebar switches from `absolute overlay` to `relative block` below board
- No glass-morphism on mobile — uses `bg-[#2d4a2d]` solid background
- Piece palette switches from vertical list to horizontal scroll (`flex flex-row overflow-x-auto`)
- Tier headers become compact inline labels
