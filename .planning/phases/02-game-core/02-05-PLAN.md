Fix the incomplete battleOutcome payload so BattleReveal renders attacker and defender pieces correctly. The server must include attacker and defender pieces in the move:result payload, and the client must transform it into the client's BattleOutcome type.
</objective>

**Problem:** Server's BattleOutcome type has `{winner, capturedPieceIds, attackerWins, attackerRevealed, defenderRevealed}` — NO attacker or defender piece data. Client's BattleOutcome type expects `{attacker: Piece, defender: Piece, attackerPosition: Position, defenderPosition: Position, result: BattleOutcomeResult}`. BattleReveal renders attacker.type and defender.type — these are undefined.

**Server types (mismatched):**
```typescript
// server/src/types/index.ts - current
export interface BattleOutcome {
  winner: 'red' | 'blue' | 'tie';
  capturedPieceIds: string[];
  attackerWins: boolean | null;
  attackerRevealed: boolean;
  defenderRevealed: boolean;
}

// client/src/store/gameStore.ts - expected
export interface BattleOutcome {
  attacker: Piece;
  defender: Piece;
  attackerPosition: Position;
  defenderPosition: Position;
  result: BattleOutcomeResult; // 'attacker_wins' | 'defender_wins' | 'tie'
}
```

**Solution:** The server's move:result should include the raw attacker and defender pieces plus from/to positions in the payload. The client transforms this into the client's BattleOutcome format.
</context>

  <name>Task 1: Server includes attacker and defender pieces in move:result payload</name>
  <files>server/src/socket/handlers/gameHandler.ts</files>
  <action>
    In the make-move handler (around line 341-346), the move:result emission currently emits:

    ```typescript
    io.to(roomId).emit('move:result', {
      move: { from, to },
      outcome: battleOutcome,
      board: room.board,
      currentTurn: room.currentTurn,
    });
    ```

    Change it to include the attacker and defender pieces and positions in the payload (outside of `outcome`, since `outcome` is the server's BattleOutcome type):

    ```typescript
    const attacker = room.board[from.row][from.col];
    const defender = target ? room.board[to.row][to.col] : null;
    io.to(roomId).emit('move:result', {
      move: { from, to },
      outcome: battleOutcome,
      attacker,
      defender,
      attackerPosition: from,
      defenderPosition: to,
      board: room.board,
      currentTurn: room.currentTurn,
    });
    ```

    Note: `target` is the defending piece at the destination — it was captured by `applyMove` and the board was already updated. So read the attacker from the board AFTER applyMove updates it (attacker moved to `to`), or capture these before applyMove. Since `applyMove` modifies room.board, `room.board[from.row][from.col]` will be null (piece already moved) and `room.board[to.row][to.col]` will be the attacker (if no battle) or null (if battle). 

    **Fix:** Read the attacker and defender pieces BEFORE calling applyMove:

    ```typescript
    const attacker = room.board[from.row][from.col];
    const defender = room.board[to.row][to.col];
    // Apply move and resolve battle
    const { room: updatedRoom, battleOutcome } = applyMove(room, from, to);
    // ... rest unchanged, but include attacker/defender in the emit:
    io.to(roomId).emit('move:result', {
      move: { from, to },
      outcome: battleOutcome,
      attacker,
      defender,
      attackerPosition: from,
      defenderPosition: to,
      board: updatedRoom.board,
      currentTurn: updatedRoom.currentTurn,
    });
    ```

    Change to transform the server payload into the client's BattleOutcome format:

    ```typescript
    const handleMoveResult = (data: { move: { from: Position; to: Position }; outcome: any; attacker: Piece | null; defender: Piece | null; attackerPosition: Position; defenderPosition: Position; board: (Piece | null)[][]; currentTurn: 'red' | 'blue' }) => {
      setBoard(data.board);
      setTurn(data.currentTurn);
      if (data.attacker && data.defender && data.outcome) {
        let result: 'attacker_wins' | 'defender_wins' | 'tie' = 'tie';
        if (data.outcome.winner === 'tie') {
          result = 'tie';
        } else if (data.outcome.attackerWins === true) {
          result = 'attacker_wins';
        } else if (data.outcome.attackerWins === false) {
          result = 'defender_wins';
        }
        setBattleOutcome({
          attacker: data.attacker,
          defender: data.defender,
          attackerPosition: data.attackerPosition,
          defenderPosition: data.defenderPosition,
          result,
        });
      }
    };
    ```

    Note: attacker and defender are both non-null when there's a battle (moving to occupied square). The if-check handles both battle and non-battle cases correctly.
