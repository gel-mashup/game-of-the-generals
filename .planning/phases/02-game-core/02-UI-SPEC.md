## Interaction Patterns

### Phase 2 — Game Core Interactions

#### 1. Piece Selection (playing phase)
- Click own piece on board → gold border highlight (`#d4a847`), valid moves shown
- Click opponent's piece → brief red flash on that piece, current selection deselects
- Click non-selectable piece (opponent's, wrong turn) → deselects current selection
- Click outside board → deselects current selection
- Gold border: 2px solid, same as palette piece selection (Phase 1)

#### 2. Turn Indicator (three-part system)
- **Header text:** "Your turn" (own turn) / "Waiting for opponent…" (opponent turn)
- **Side glow:** current player's side in header area highlights with player color at 20% opacity
- **Board tint:** subtle color overlay on current player's side of board at 5% opacity

#### 3. Valid Move Highlighting
- Green-tinted squares (`rgba(74, 124, 74, 0.5)`) show when piece selected
- Only adjacent orthogonal empty squares highlighted
- No indication of invalid destinations — only valid options
- Pieces on own side cannot be moved to (blocked, no highlight)

#### 4. Move Execution
- Click highlighted valid square → piece moves
- Client optimistically updates, reverts if server rejects
- Server validates: ownership, turn, adjacency, destination rules

#### 5. Invalid Click Feedback
- Opponent's piece clicked: brief red flash (200ms) — click registered but action not allowed
- Not your turn + try to move: no response (passive rejection, no visual change)
- Invalid board square (deployment): silent rejection — no message, no animation

#### 6. Battle Reveal
- **Inline on board:** pieces slide together (not modal overlay)
- **Reveal duration:** ~1 second
- **Higher rank wins:** losing piece removed with fade-out, winning piece stays
- **Equal rank tie:** explosion/spark effect on both pieces, both fade out simultaneously
- Explosion effect: radial gradient burst in orange-red (`rgba(255, 100, 50, 0.8)`) + 6-8 spark particles radiating outward, CSS animation, ~600ms duration
- Turn after battle: standard rules — turn switches to other player automatically

#### 7. Auto-Deploy Button
- Position: below piece palette, secondary style (not accent gold)
- Placement: all 21 pieces placed instantly, no animation
- Post-auto-deploy: player can still manually rearrange any pieces
- Re-randomize: clicking again re-randomizes all placements
- No auto-select: does not select a piece type — player picks from palette manually

#### 8. Ready State
- Button: accent gold (`#d4a847`) background, "Ready" label
- Enabled only when all 21 pieces placed
- Once clicked: button shows "Ready ✓", all placed pieces lock (no further rearrangement)
- Opponent sees "Opponent Ready" indicator in header
- Once both ready: 3-second countdown "Game starting in 3… 2… 1…" → game:started
- Countdown cannot be interrupted — no unready possible during countdown

#### 9. Deployment Zone Enforcement
- **Zone highlight:** valid deployment zone highlights in green (`rgba(74, 124, 74, 0.35)`) when a palette piece type is selected
- **Invalid square click:** silent rejection — piece doesn't place, no message, no animation
- **Rearrangement:** can freely move placed pieces within deployment zone until Ready clicked
