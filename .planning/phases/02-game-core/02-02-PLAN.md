# Socket event contracts from 02-RESEARCH.md

Client → Server:
- deploy-piece: { pieceId: string, row: number, col: number }
- auto-deploy: no payload
- ready: no payload
- make-move: { from: { row, col }, to: { row, col } }

Server → Client:
- game:started: { board, currentTurn: 'red', status: 'deploying' }
- piece:deployed: { piece, row, col, deployedCount, board, autoDeployComplete?: boolean }
- player:ready: { playerId }
- deploy:complete: { board, currentTurn: 'red' }
- move:result: { move: { from, to }, outcome, board, currentTurn }
- countdown:update: { seconds: number }
- error: { message: string }
</context>

  <action>
    Create `server/src/socket/handlers/gameHandler.ts` that exports a function `gameHandler(io: Server, socket: Socket)`.

    The `rooms` Map must be shared between roomHandler and gameHandler. Move it to a shared module `server/src/socket/rooms.ts` that both handlers import.

    Implement the following socket event handlers:

    1. **`game:started`** — auto-emitted when second player joins (or bot joins):
       - Transition room.status to 'deploying'
       - Initialize room.deployedPieces = { red: new Set(), blue: new Set() }
       - Initialize room.readyPlayers = new Set()
       - Emit `game:started` to ALL players in room with { board, currentTurn: 'red', status: 'deploying' }
       - For bot games: bot auto-deploys via `auto-deploy` event

    2. **`deploy-piece`** — deploy a single piece:
       - Payload: { pieceId: string, row: number, col: number }
       - Validate: use engine.isValidDeployment(), check player not already ready
       - If valid: add piece to room.board, add pieceId to room.deployedPieces[playerSide]
       - Broadcast `piece:deployed` to ALL in room: { piece, row, col, deployedCount, board }
       - If invalid: emit `error` only to sender

    3. **`auto-deploy`** — random deployment:
       - Validate: game status is 'deploying', player not already ready
       - Use engine.generateAutoDeploy(playerSide) to get positions
       - For each piece type in the map: create Piece object with new ID, place on board
       - Add all pieceIds to room.deployedPieces[playerSide]
       - Broadcast `piece:deployed` (multiple times, once per piece) with autoDeployComplete: true on LAST piece
       - For bot: emit this event on bot's socket automatically when game:started received

    4. **`ready`** — signal deployment complete:
       - Validate: game status is 'deploying', player not already ready, player has 21 pieces deployed
       - Add playerId to room.readyPlayers
       - Broadcast `player:ready` to ALL in room: { playerId }
       - If room.readyPlayers.size === 2: start 3-second countdown
         - Emit `countdown:update` with { seconds: 3 }
         - setTimeout → { seconds: 2 } → setTimeout → { seconds: 1 } → setTimeout → `deploy:complete`
       - For bot games: bot is auto-ready when human player clicks Ready

    5. **`make-move`** — execute a move:
       - Payload: { from: { row, col }, to: { row, col } }
       - Validate: use engine.canMove(), verify it's player's turn, player owns piece at 'from'
       - If valid: use engine.applyMove() to execute move and get battleOutcome
       - Toggle room.currentTurn to other player
       - Broadcast `move:result` to ALL in room: { move: { from, to }, outcome: battleOutcome, board, currentTurn }
       - If invalid: emit `error` only to sender

    Helper: getPlayerSide(socketId: string, room: Room): 'red' | 'blue' | null
    - Find player in room.players by id, return their side

    The gameHandler must access the shared `rooms` Map. Move it to `server/src/socket/rooms.ts`:
    ```typescript
    import type { Room } from '../types';
    export const rooms = new Map<string, Room>();
    ```
    Then import and use it in both roomHandler and gameHandler.
  </action>
