  <name>Fix rematch:ready handler to show opponent wants rematch prompt</name>
  <files>client/src/app/game/[roomId]/page.tsx</files>
  <action>
    In the `rematch:ready` socket handler (around line 205), update the body to handle both `bothReady` cases:

    Replace the current handler:
    ```typescript
    socket.on('rematch:ready', (data: { bothReady: boolean }) => {
      if (data.bothReady) {
        // Both confirmed — server will send rematch:confirmed next
        setOpponentWantsRematch(false);
      }
    });
    ```

    With:
    ```typescript
    socket.on('rematch:ready', (data: { bothReady: boolean }) => {
      if (data.bothReady) {
        // Both confirmed — server will send rematch:confirmed next
        setOpponentWantsRematch(false);
      } else {
        // Opponent wants rematch — show prompt
        // The socket.id is this client's socket; the requesting player is whoever sent the event
        // Since this event is broadcast to ALL players, self-sends are possible in some configs.
        // We use the socket itself (not data) to identify self — if the sender were self,
        // the client's own click handler would have already set iWantRematch.
        // So any rematch:ready with bothReady=false means the OPPONENT wants rematch.
        setOpponentWantsRematch(true);
      }
    });
    ```

    The key insight: the socket handler runs for ALL incoming `rematch:ready` events. If `bothReady=false`, the opponent requested but this client hasn't clicked yet. So set `opponentWantsRematch=true`.
  </action>
