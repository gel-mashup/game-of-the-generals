# Plan 01: Fix game not starting after room join

## Objective
Fix the issue where bot pieces are not placed on the board after creating and joining a bot game room.

## Problem Analysis
Server emits `game:started` and `piece:deployed` events synchronously during room creation, BEFORE the client has navigated to the game page and set up socket listeners. Socket.io queues events but by the time the game page mounts, it misses all events.

## Solution
Add a `sync-game-state` event mechanism:

1. **Server side**: Add `sync-game-state` handler in `gameHandler.ts` that:
   - Finds the room the socket belongs to
   - Emits `game:started` with current board state, turn, and status

2. **Client side**: Game page emits `sync-game-state` on mount to request current state

## Files to Modify
- `server/src/socket/handlers/gameHandler.ts` - Add `sync-game-state` handler
- `client/src/app/game/[roomId]/page.tsx` - Add `socket.emit('sync-game-state')` on mount

## Verification
1. Create bot room → bot auto-deploys on server
2. Client joins game page → requests sync
3. Server responds with state → game enters "deploying"
4. Player auto-deploys and clicks Ready → countdown → "playing"
